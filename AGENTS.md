# Agent Guide

This repo is a standalone Expo/RN library for `@r4z33n4l1/whats-new-sheet`.

## Shape Of The Project

- `src/` is the package source. Edit this first.
- `example/` is the Expo Router playground.
- `lib/` is generated by Bob. Do not edit it manually.
- `README.md` is the public npm/GitHub landing page.
- `example/README.md` explains the playground.

## Product Boundaries

Keep the package lightweight and data-first.

In scope:

- `WhatsNewSheet` presentation.
- Typed announcement/page data.
- Seen-state helpers through `WhatsNewStorageAdapter`.
- List, image, and custom pages.
- Expo Router examples in the playground.

Out of scope for v1:

- Backend/admin service.
- AsyncStorage as a dependency.
- Expo Video or other optional native modules.
- A required navigation library.
- Ritual-specific imports or design tokens in package source.

## Commands

```sh
corepack yarn install
corepack yarn typecheck
corepack yarn prepare
npm pack --dry-run
corepack yarn example start
```

Run `npm pack --dry-run` before claiming the package is publish-ready. The tarball should not include `example/`, `node_modules/`, secrets, tests, or generated app artifacts.

## Design Guidance

The default visual tone should feel closer to a premium app release note than a generic onboarding carousel:

- Short badge: "What's New".
- One strong title.
- Optional short body copy.
- Three concise feature rows when possible.
- Warm neutral surfaces.
- One primary CTA.

Prefer improving the core data model and defaults over adding many one-off props. Add API only when it helps multiple apps.

## Release Notes

For every behavior/API change, update `CHANGELOG.md`.

Do not publish from an agent session unless the user explicitly asks. Preparing a tarball is fine; publishing is externally visible.
